# Лабораторная работа №15-16. Navigation in Jetpack Compose

## Краткое описание приложения
Мобильное приложение для студентов, позволяющее отслеживать учебные дисциплины в текущем семестре. Для каждого предмета отображается название, преподаватель, количество баллов, текущая оценка и подробное описание. Приложение поддерживает навигацию между экранами: список дисциплин, детальная информация, профиль студента и настройки.

## Список реализованных экранов

- **Главный экран (HomeScreen)** – отображает список всех дисциплин в виде карточек.
- **Экран деталей (DetailsScreen)** – показывает подробную информацию о выбранной дисциплине (описание, преподаватель, баллы, оценка).
- **Расписание (Schedule)** — список занятий, сгруппированных по дням недели. Каждое занятие отображает название предмета, время, аудиторию и преподавателя.
- **Детали занятия (LessonDetail)** — подробная информация о конкретном занятии (день, время, аудитория, преподаватель).
- **Экран профиля (ProfileScreen)** – содержит данные студента (ФИО, группа, факультет, контакты, статистика).
- **Экран настроек (SettingsScreen)** – позволяет включать/отключать уведомления, переключать тему и просматривать информацию о приложении.

## Используемые технологии
- **Kotlin** – язык разработки.
- **Jetpack Compose** – современный UI-фреймворк для создания интерфейсов.
- **Navigation Compose** – библиотека для управления навигацией между экранами.

## Контрольные вопросы 

## 1. Что такое NavController и для чего он используется?
`NavController` управляет навигацией между экранами.  
`rememberNavController()` сохраняет его при рекомпозиции, иначе контроллер будет пересоздаваться.

## 2. Как передать параметр в маршрут навигации?
В маршруте указывается плейсхолдер: `"details/{subjectId}"`.  
В `composable` добавляется `arguments = listOf(navArgument("subjectId") { type = NavType.StringType })`.  
Значение извлекается: `backStackEntry.arguments?.getString("subjectId")`.  
**Разница:** обязательный параметр не имеет значения по умолчанию; опциональный – имеет `defaultValue`.

## 3. Зачем использовать sealed class для маршрутов?
- Типобезопасность (исключает опечатки в строках).  
- Автодополнение в IDE.  
- Пример ошибки: `navController.navigate("profle")` вместо `"profile".

## 4. Что такое Back Stack и как им управлять?
Back Stack – стек экранов в порядке посещения.  
Схема для `Home → Profile → Settings`: `[Home, Profile, Settings]`.  
При вызове `popBackStack()` на экране `Settings` он удаляется, показывается `Profile`.

## 5. Как работает startDestination в NavHost?
Определяет экран, который отображается первым при запуске.  
Можно изменить динамически, передав другое значение в `NavHost`, но обычно startDestination фиксирован.

## 6. Что произойдёт, если навигаровать на несуществующий маршрут?
`NavHost` не найдёт соответствующий `composable` и выбросит `IllegalArgumentException`.  
Обработать можно `try-catch`.

## 7. Зачем нужен параметр launchSingleTop в навигации?
Предотвращает создание дубликата экрана в верхушке стека.  
Пример проблемы: при многократном клике на кнопку "Профиль" создаётся несколько одинаковых экранов.  
`launchSingleTop` оставляет только один экземпляр.
